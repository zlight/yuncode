# 列表数据服务器请求逻辑实现计划

## Phase 1: 统一列表请求与状态体验 ✅
- [x] 为前端列表类页面建立统一的服务器请求流程，覆盖加载、刷新、错误和空状态
- [x] 保持深色高端云平台视觉风格，使用青蓝色强调、半透明深色卡片和清晰加载反馈
- [x] 将列表筛选、搜索、排序和分页行为与服务器端请求结果联动
- [x] 确保双语提示、按钮和状态信息默认中文显示并可切换英文

## Phase 2: 服务器目录与套餐列表请求化 ✅
- [x] 将服务器目录中的套餐、地区、节点、库存和价格列表改为通过服务器请求获取
- [x] 支持查询参数驱动的地区预选、筛选条件变更后重新请求、重置筛选后重新加载
- [x] 保留购买流程、库存扣减、订单创建和账户联动逻辑
- [x] 为网络异常、无结果、加载中和刷新成功场景提供明确反馈

## Phase 3: 控制台列表与管理列表请求化 ✅
- [x] 将控制台中的实例列表、账单列表和可管理资源列表改为通过服务器请求获取
- [x] 让搜索、地区筛选、自动续费、刷新和管理页选择行为与服务器端数据保持一致
- [x] 确保用户只能请求并查看自己的资源、订单和实例
- [x] 验证服务器目录、购买后控制台同步、筛选搜索和刷新流程稳定可用
